# Python编码规范  

在编写Python代码时, 你需要遵循以下编码规范:  

**语言级别:** `Python 3.12`.  
**类型注解:** 任何变量, 返回值等都需要类型注解, 当成可多类型的强类型语言进行编码  
**编码规范:** 遵循PEP8. 我在PyCharm最严格模式下, 你需要保证不能有任何Warning. 采用现代的写法, 包括不要从`__future__`导入, 不要使用`Optional`等而是采用3.10+的新类型注解等, 不要使用`pass`而是使用`...`. 善于使用`match-case`, 海象运算符(仅当有必要时), `lambda`, `zip`, `map`等现代优雅的`Pythonic`的写法; 包括使用抽象类,`@staticmethod`, `@classmethod`, `@dataclass`, `@property`等类型注解  
**文档字符串:** 在文件开头, 任何的函数, 类(包括枚举等数据结构)内都要有遵循以下规范的文档字符串(包括内部子函数). 代码即文档的编码规范: 代码需要尽可能的自描述, 易懂, 减少非文档字符串之外的注释内容.  
文件开头:  

```python
r"""
文件功能描述, 使用中文

:file: 文件名, 从项目目录开始的相对路径  
:author: 大师
:time: 时间, 如2025-12-22. 更新为最新时间.  
"""
```

函数及方法(内部)  

```python
r"""
函数功能描述, 使用中文
:param 参数名: 参数说明
:return 返回值类型: 返回值说明
:raise 异常名: 异常说明 
"""
```

类:  

```python
r"""
类描述, 中文
"""
```
